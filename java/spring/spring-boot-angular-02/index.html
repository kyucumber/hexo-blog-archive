Angular 프로젝트 설정하기

프로젝트를 Backend와 Front 분리하는게 좋겠지만 그냥 spring 프로젝트랑 통채로 묶어서 진행한다.

이걸 볼 사람은 없겠지만 Front랑 Backend 프로젝트를 분리하시길.. 일단 angular-cli를 설치하자.

그리고 스프링 프로젝트 안에서

/src/main 이 경로상에서 아래의 커맨드를 실행시키자.

그리고 Spring Boot 배포 설정을 맞추기 위해서 angular-cli.json 구성에 몇 가지를 추가하자.

별건 없고 project.name, outDir 두개를 수정하자. 프로젝트 이름은 프로젝트 설명으로, outDir은 dist/static으로 변경하자.

  • angular-cli.json

build.gradle에 아래 내용 추가

sourceSets에 자바랑 코틀린은 코틀린과 자바를 섞어쓰려고 넣은 설정이고 여기서는 main.resources.srcDirs = ["$staticDir/dist", "$projectDir/src/main/resources"]만 넣어주면 된다.

  • build.gradle

표준 processResources 태스크에 buildAngular 태스크 종속성을 추가하고..

installAngular에서 npm install, buildAngular에서 ng build를 실행하고 빌드된 결과는 위 angular-cli.json 설정에서 outDir로 설정한 webapp/dist/static에 저장되게 된다.

대충 npm install을 해주고 angular를 /dist 경로에 빌드하고 스프링 프로젝트가 빌드된 index.html을 바라보게 하는 설정 같은 느낌.
