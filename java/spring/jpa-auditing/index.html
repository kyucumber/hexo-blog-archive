Jpa Auditing 적용하기

2018-09-01 00:00:00
Backend>Java>Spring

JPA를 사용하면 중복되는 부분들이 많이 발생한다. 등록 시간, 수정 시간, id 등등.. 그런 부분을 Auditing Entity를 통해 중복을 제거하고 자동화 할 수 있다.

그리고, @EnableJpaAuditing 어노테이션 추가

위와같이 설정하면 createDate와 modifiedDate에 대해서 자동으로 변경해주게 된다.

createdBy나 lastModifiedBy도 자동으로 설정할 수 있는데, 이는 아래와 같은 추가적인 설정을 더 넣어야 한다.

우선 설정 파일에 auditorAwareRef를 추가한다.

그리고 누구에 의해서 작성, 수정되었는지를 가져올 securityAuditorAware를 생성하자.

일반적으로 세션을 사용하는 로그인의 경우에는 적용되지 않는 방법이고 스프링 시큐리티를 사용한 경우 위와 같은 방법으로 가져올 수 있다. 세션 사용 기반에서는 세션이 아니라 스레드 로컬에 저장하도록 하는식으로 해서 구현할 수 있을것 같은데.. 안해봐서 잘 모르겠다.

그리고 추가적으로 LocalDateTime을 사용하기 때문에 Serialize, DeSerialize 할때 문제가 발생할 가능성이 높다. 커스텀 Serializer나 DeSerializer를 만들어서 등록해서 사용하자.
