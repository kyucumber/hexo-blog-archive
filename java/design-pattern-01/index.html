Java 디자인 패턴 01 - Abstract Factory Pattern

2018-04-17 21:30:00
Java>DesignPattern

추상 팩토리 패턴(Abstract Factory Pattern)

Provide an interface for creating families of related or dependent objects without specifying their concrete classes.

구체적인 클래스를 지정하지 않고 관련 객체 또는 종속 객체의 패밀리를 작성하기 위한 인터페이스를 제공합니다.

상세화된 서브클래스를 정의하지 않고도 서로 관련성이 있거나 독립적인 여러 객체의 군을 생성하기 위한 인터페이스를 제공합니다.

Factory라는 말에서 유추할 수 있지만 어떤 객체들을 생성하기 위한 인터페이스를 제공하는 패턴이라고 합니다.

설명이 어려운데 개인적으로 이해한 걸 풀어서 써 보면 관련이 있는 객체들을 생성하기 위한 인터페이스(Abstract Factory)를 생성하고 이 팩토리를 이용해 객체 생성을 캡슐화하고, 관련된 클래스들의 생성을 위임하는 패턴인것 같습니다.

설명만 듣고서는 잘 이해가 안돼서 java-design pattern github repo의 예제를 참고했습니다.

성, 왕, 군대 세개의 인터페이스가 있다고 가정합니다.

사람들이 세운 나라와 엘프들이 세운 나라에는 각각 성, 왕, 군대가 있다고 한다면..

HumanCastle, HumanKing, HumanArmy와 ElfCastle, ElfKing, ElfArmy를 생성할 수 있습니다.

여기서 위의 Human과 관련된 객체들(HumanKing, HumanCastle, HumanArmy)를 생성하기 위한 HumanFactory를 정의하고 HumanKindomFactory에서 Human과 관련된 객체들을 생성해 줄 수 있습니다.

위에서 설명했던 것 처럼 구체적인 클래스를 직접 지정하지 않고 관련 객체 또는 종속 객체의 패밀리를 작성하기 위한 HumanKindomFactory를 생성했다고 할 수 있습니다.

이와같이 Elf와 관련된 Kindom의 객체들을 생성해주는 ElfKindomFactory도 생성할 수 있습니다.
