Java 디자인 패턴 01 - Abstract Factory Pattern

2018-04-17 21:30:00
Backend>Java>DesignPattern

추상 팩토리 패턴(Abstract Factory Pattern)

Provide an interface for creating families of related or dependent objects without specifying their concrete classes.

구체적인 클래스를 지정하지 않고 관련 객체 또는 종속 객체의 패밀리를 작성하기 위한 인터페이스를 제공합니다.

상세화된 서브클래스를 정의하지 않고도 서로 관련성이 있거나 독립적인 여러 객체의 군을 생성하기 위한 인터페이스를 제공합니다.

Factory라는 말에서 유추할 수 있지만 어떤 객체들을 생성하기 위한 인터페이스를 제공하는 패턴이라고 합니다.

설명이 어려운데 개인적으로 이해한 걸 풀어서 써 보면 관련이 있는 객체들을 생성하기 위한 인터페이스(Abstract Factory)를 생성하고 이 팩토리를 이용해 객체 생성을 캡슐화하고, 관련된 클래스들의 생성을 위임하는 패턴인것 같습니다.

Factory method 패턴과 Abstract Factory 패턴은 유사한 구조를 가지고 있습니다. 제가 이해한게 맞는진 모르겠지만.. 가장 큰 차이점은 Factory method에서는 하나의 객체를 생성하지만, Abstract Factory에서는 어떤 관련성이 있는 여러 객체의 그룹을 생성합니다.

즉 생성할 객체가 한가지다. 하면 Factory method, 어떤 유사한 객체의 군(여러 객체)을 묶어서를 생성할 때는 Abstract Factory 패턴을 적용하면 될 것 같은데.. 잘 모르겠네요

설명만 듣고서는 잘 이해가 안돼서 java-design pattern github repo의 예제를 참고했습니다.

Example

전체 구조

Castle, King, Army 인터페이스, 이는 하나의 Kindom 그룹을 형성한다.

Castle(interface)

​ ┗ ElfCastle

​ ┗ HumanCastle

King(interface)

​ ┗ ElfKing

​ ┗ HumanKing

Army(interface)

​ ┗ ElfArmy

​ ┗ HumanArmy

Castle, King, Army를 그룹으로 묶어 생성할 수 있는 Factory

KingdomFactory(interface)

​ Castle createCastle()

​ King createKing()

​ Army createArmy()

​ ┗ ElfKingdomFactory

​ ┗ HumanKingdomFactory

enum을 이용해 Factory method 패턴으로 Factory(Elf, Human..) 생성

FactoryMaker(static class)

​ makeFactory(KingdomType)

Kindom 그룹의 클래스(Castle, King, Army)

Castle과 King Army는 하나의 Kingdom 그룹에 속해야 합니다.

KindomFactory에서는 그 서브클래스들을 그룹으로 묶어 생성하기 위한 인터페이스를 제공합니다.

예제에서는 ElfCastle, HumanCastle …을 생성합니다.

사람들이 세운 나라와 엘프들이 세운 나라에는 각각 성, 왕, 군대가 있다고 한다면..

HumanCastle, HumanKing, HumanArmy와 ElfCastle, ElfKing, ElfArmy를 생성할 수 있습니다.

Kindom 그룹 객체들을 생성할 Factory 클래스

여기서 위의 Human과 관련된 객체들(HumanKing, HumanCastle, HumanArmy)를 생성하기 위한 HumanFactory를 정의하고 HumanKindomFactory에서 Human과 관련된 객체들을 생성해 줄 수 있습니다.

위에서 설명했던 것 처럼 구체적인 클래스를 직접 지정하지 않고 관련 객체 또는 종속 객체의 패밀리를 작성하기 위한 HumanKindomFactory를 생성했다고 할 수 있습니다.

이와같이 Elf와 관련된 Kindom의 객체들을 생성해주는 ElfKindomFactory도 생성할 수 있습니다.

아래처럼 KindomFactory에 ElfKindomFactory를 할당해주고, 구현된 create() 메소드들을 이용해 Elf와 관련된 객체들을 가져올 수 있습니다.

위에서는 직접 사용자가 어떤 Factory를 사용할지 명시적으로 적어주고 있는데, enum을 이용한 Factory method 패턴을 구현해 KingdomFactory를 얻어오는 부분을 아래처럼 변경할 수 있습니다.

Abstract Factory 패턴에서는 위처럼 팩토리 생성만 다르게 하고, 나머지 castle, king, army를 사용하는 건 동일하게 동작하게 됩니다. 이해한 게 맞는지는 잘 모르겠네요
