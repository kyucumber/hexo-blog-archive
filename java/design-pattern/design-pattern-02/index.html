Java 디자인 패턴 02 - Factory Method Pattern

2018-04-23 21:30:00
Backend>Java>DesignPattern

팩토리 메소드 패턴(Factory Method Pattern)

Define an interface for creating an object, but let subclasses decide which class to instantiate. Factory Method lets a class defer instantiation to subclasses.

객체를 생성하기 위한 인터페이스(Factory)를 정의하고 객체를 만들어내는 부분을 서브클래스에 위임하는 패턴입니다.

Abstract Factory 패턴에서 Factory Method 패턴을 활용하는 경우가 있기 때문에 Abstract Factory 이전에 다루었어야 하는데..

Example

Weapon과 Weapon을 만드는 Blacksmith를 예로 들어보겠습니다.

전체 구조

Weapon(interface)

​ ┗ ElfWeapon

​ ┗ OrcWeapon

Blacksmith(interface)

​ ┗ ElfBlacksmith

​ ┗ OrcBlacksmith

Weapon은 ElfWeapon이 있고, OrcWeapon이 있습니다.

각각의 Weapon을 만드는 Factory인 ElfBlacksmith, OrcBlacksmith가 존재합니다.

Factory에서 생성할 Weapon 클래스

Factory method 패턴을 이용해 Weapon 클래스를 생성합니다. Weapon은 ElfWeapon, OrcWeapon이 있고 HumanWeapon 등 더 확장될 수 있습니다.

Weapon을 생성할 Factory 클래스

이제 각각의 Elf, Orc에 해당하는 Weapon을 만드는 각각의 Factory를 기본 Factory 인터페이스를 구현해 정의합니다.

이제 ElfWeapon과 OrcWeapon을 만들 Factory인 ElfBlacksmith와 OrcBlacksmith가 생성되었습니다.

Factory를 이용한 Weapon의 생성

이제 ElfBlacksmith를 이용해 Weapon 클래스를 생성합니다. 객체의 생성은 Factory에서 이루어지게 됩니다.

팩토리 메소드 패턴을 이용하면 직접 코드로 객체를 생성하는것을 방지하고 객체 생성을 서브 클래스에 위임하기 때문에 의존성을 제거해 클래스간의 결합도가 낮아진다고 합니다.
