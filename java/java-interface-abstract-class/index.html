Java의 인터페이스와 추상 클래스

2018-04-15 21:30:00
Backend>Java

인터페이스와 추상 클래스를 자주 사용하기는 하는데, 항상 조금씩 혼동이 와서 이번 기회에 정리합니다.

우선 클래스와 추상 클래스, 인터페이스는 아래와 같은 외형적인 차이점이 있습니다.

클래스

그냥 일반적인 클래스의 기능을 한다.

추상 클래스

abstract 키워드가 붙거나 클래스 내 추상 메소드가 하나 이상 포함되는 클래스

만약 추상 클래스를 상속한다면, 추상 클래스에 존재하는 추상 메소드를 반드시 구현해주어야 하며 extends 키워드를 사용합니다.

제일 위의 추상 메소드가 없는 추상 클래스는 사실 그냥 클래스와 같은 것 같은데..

인터페이스

추상 메소드만을 가지며, 메소드의 껍데기만 존재한다. 서브클래스는 무조건 모든 메소드들을 구현해야하며 implements 키워드를 사용합니다.

Java 8부터 인터페이스도 default를 통해서 default 메소드의 구현이 가능해졌다.

default 메소드가 추가되면서 하나의 문제가 생겼는데.. 기존에 클래스는 다중 상속이 되지 않습니다.

그런데 인터페이스에 default 메소드가 추가되면서 아래와 같은 문제가 생깁니다.

두개의 인터페이스를 구현하는 서브클래스에서 만약 default 메소드 이름 두개가 같다면? 다중 상속의 모호함이 인터페이스에서 일어나게 됩니다.

위에서는 두 인터페이스 전부 globalGreeting() 메소드를 default로 가지기 때문에 구현하는 두개의 인터페이스를 구현하는 클래스에서 globalGreeting()이 모호하게 되고, 서브클래스에서는 재정의를 하지 않으면 에러가 발생합니다.

default의 이름이 겹치는 경우에는 default 메소드여도 구현을 필수적으로 하도록 설정되어 있는거 같습니다.

인터페이스와 추상 클래스의 차이점

위에서 다룬 외형적인 차이점도 있고.. 아직 개발 경험이 많지 않다 보니 유사하게 사용하고 있는 것 같지만 추상클래스와 인터페이스는 존재하는 목적이 조금 다른것 같습니다.

차이점을 간략하게 말하면 아래와 같은 내용들 정도가 있을 것 같습니다.

  • 추상 클래스는 상속을 통해 부모 클래스의 기능을 이용, 확장하기 위해 제공되며 다중 상속이 불가능하다.
  • 인터페이스는 빈 껍데기 형태로 구현할 메소드를 정의하고, 강제해서 서브클래스들에 같은 동작을 보장하기 위해 제공되며 다중 상속이 가능하다.

인터페이스에 발생하는 중복 제거

예전에 인터페이스만을 이용해 요구되는 명세를 개발하면서 인터페이스의 구현 메소드에서 중복이 발생하는 경우가 꽤 많았는데 그때 추상클래스를 이용해 중복을 제거할 수 있을것 같습니다. 실제로 적용한 적은 없고 생각만 해본 케이스지만.

아래의 예에서는 Animal이란 인터페이스를 만들고, walk라는 구현해야 할 메소드를 생성했습니다.

위의 코드에서 토끼와 사자는 4족 보행 동물로 아래같이 구현하면 다른 4족 보행 동물에 대해서도 walk를 다 구현해줘야하는 중복이 발생합니다.

4족 보행들의 걷는 walk 메소드는 미리 구현을 해 두고 싶지만, Animal 인터페이스에 default로 두면 2족 보행하는 서브클래스들에 대해서도 일괄적으로 적용되기 때문에 문제가 발생합니다.

사실 인터페이스를 2개로 분리할 수도 있지만.. 아래처럼 2족 보행 동물에 대한 TwoLegAnimal 추상 클래스, 4족 보행에 대한 FourLegAnimal 추상 클래스를 추가하면 번거롭긴 하지만 walk를 분리하면서도 하나의 인터페이스를 이용해 아래처럼 중복된 부분을 제거할 수 있습니다.
