자료구조 - Graph

2018-04-16 21:30:00
CS>Datastructure

그래프는 자료구조의 일종입니다. 실생활에서 예를 들자면 지하철에서 역은 정점(Vertex), 간선은 역 사이, 도로는 간선, 사거리나 교차로가 정점, 페이스북을 예로 들자면 사람이 정점, 친구 관계 사이에 간선이 연결되게 됩니다.

  • 정점(Node, Vertex)
  • 간선(Edge) 정점간의 관계를 나타낸다.

G = (V, E) 형태로 나타냅니다.

경로(Path)

정점 A에서 B로 가는 경로

A - C - D - E - B

A - B

A - C - B

A - C - E - B

사이클(Cycle)

정점 A에서 A로 다시 돌아오는 경로

A -> C -> B -> A

A -> C -> E -> B -> A

A -> C -> D -> E -> B -> A

단순 경로와 단순 사이클

경로/사이클에서 같은 정점을 두번 이상 반복하지 않는 경로와 사이클 특별한 말이 없으면 일반적으로 사이클과 경로는 단순 경로/사이클을 말합니다.

방향 있는 그래프(Directed Graph)

방향이 있는 그래프, 방향이 없다면 양방향으로 연결되게 되고 아래의 Directed Graph에서는 A -> C로 갈 수 있고 C -> A로는 갈 수 없습니다.

양방향 그래프(Bidirection Graph) 라고도 한다. 일반적으로 자료구조 형태로 저장할때는 방향있는 그래프로 바꾸어서 양방향인 경우 두 간선을 모두 연결하는 방법으로 구현하게 됩니다.

간선이 여러개인 경우(Multiple Edge)

두 정점 사이에 간선이 여러개인 경우

루프(Loop)

간선의 양 끝 점이 같은 경우, A -> A

가중치(Weight)

간선을 지나갈때 필요한 시간, 거리 등등을 나타낸다. 보통 문제는 이 가중치를 바탕으로 최단거리를 구하거나 하는 경우가 많은데 가중치가 없는 그래프에서는 가중치를 1로 두고 문제를 풀면 됩니다.

아래에서 간선별로 표시된 숫자가 가중치에 해당합니다.

차수(Degree)

정점과 연결된 간선의 갯수를 말합니다.

정점으로 들어오는 간선 : indegree

정점에서 나가는 간선 : outdegree

아래에서는 A는 outdegree가 2, indegree가 0, C는 outdegree가 3, indegree가 1이다.

인접 리스트(Adjacency list)를 통한 그래프의 저장

인접 리스트는 자바에서 사용하는 ArrayList를 이용해 연결되는 간선을 저장합니다.

A[1] : 2, 5

A[2] : 1, 3, 4, 5

A[3] : 2, 4

A[4] : 3, 5, 2, 6

A[5] : 1, 2, 4

A[6] : 4

위처럼 저장이 되는데, 이러면 간선의 갯수만큼의 공간만을 차지합니다. 인접 행렬은 정점의 N^2의 공간 복잡도를 가지지만 인접 리스트는 간선 갯수만큼의(O(E)) 공간을 차지합니다.

그래프의 탐색 - 깊이 우선 탐색(DFS)

Depth First Search 깊이 우선, 최대한 깊숙히 탐색 Stack 사용

스택을 이용해 갈 수 있는 만큼 최대한 많이 깊게 가고, 갈 수 없으면 이전에 탐색한 정점으로 돌아가 탐색을 진행한다.

하나의 그래프에서 탐색을 하기 위해 아래와 같은 공간이 필요하다.

  • 방문 여부를 저장할 check 배열
  • 스택(재귀 호출 사용)

Step 1

시작 정점을 1로 가정하고, 위의 그래프에서 DFS를 시작하면 스택에 정점 1을 넣고, 해당 정점을 방문했으므로 check 배열의 1 요소를 true로 변경합니다.

index 1 2 3 3 5 6
check[] true false false false false false

방문 순서 : 1

Stack : 1

Step 2

그 다음, 1정점과 연결된 2 정점을 방문합니다.

현재 스택 최상단의 요소는 2, check 배열에 2의 값도 방문했음을 체크합니다.

index 1 2 3 3 5 6
check[] true true false false false false

방문 순서 : 1 2

Stack 1, 2

Step 3

다음은 정점 2와 연결된 정점 3을 방문합니다. 마찬가지로 방문 여부를 체크합니다.

i 1 2 3 4 5 6
check[i] true true true false false false

방문 순서 : 1 2 3

Stack : 1 2 3

Step 4

다음 정점 3과 연결된 정점 4를 방문합니다. 마찬가지로 방문 여부 체크

i 1 2 3 4 5 6
check[i] true true true true false false

방문 순서 : 1 2 3 4

Stack : 1 2 3 4

Step 5

다음은 정점 4와 연결된 정점 5를 방문합니다. 마찬가지로 방문 여부 체크

1 2 3 4 5 6
check[i] true true true true true False

방문 순서 : 1 2 3 4 5

Stack : 1 2 3 4 5

Step 6

그 다음에 정점 5에서 다음으로 갈 수 있는 정점이 없으므로, 스택에서 정점 5를 제거하고 정점 4로 돌아간다.

i 1 2 3 4 5 6
check[i] true true true true true false

방문 순서 : 1 2 3 4 5

Stack : 1 2 3 4

Step 7

다음 정점 4에서 갈 수 있는 정점 6으로 이동합니다.

i 1 2 3 4 5 6
check[i] true true true true true true

방문 순서 : 1 2 3 4 5 6

Stack : 1 2 3 4 6

Step 8

이제 6에서 갈 수 있는 곳이 없고, check 배열도 모두 채워졌기 때문에 스택에서 하나씩 요소를 제거하면 됩니다.

Stack : 1 2 3 4

Stack : 1 2 3

Stack : 1 2

Stack : 1

이제 스택에서 모든 정점이 빠지고, DFS 탐색이 종료됩니다.

위에서 봤듯, 계속 depth를 따라 깊이 우선으로 탐색을 하고, 탐색할 정점이 없으면 이전 정점으로 돌아갑니다. java 코드로 작성하면 재귀호출을 이용해서 아래처럼 구현할 수 있습니다.

그래프의 탐색 - 너비 우선 탐색(BFS)

Breadth First Search 너비 우선 탐색, 너비 우선은 Queue를 사용합니다.

모든 가중치가 1인 경우 최단거리를 찾는다. 가중치가 1일 때 BFS를 이용해 최단거리를 찾을 수 있다. 큐를 이용해 지금 위치에서 갈 수 있는 정점을 모두 큐에 넣는 방식

Step 1

정점 1에서 시작합니다.

큐에 정점 1을 넣고, 정점 1 방문을 체크합니다.

그리고 현재 정점 1에서 인접한 2, 5 를 큐에 넣고 2, 5방문 여부를 체크합니다.

i 1 2 3 4 5 6
check[i] true true false false true False

방문 순서 : 1

Queue : 1, 2, 5

Step 2

큐에서 정점 1을 제거하고, 정점 2를 방문합니다.

그리고 인접한 3을 큐에 넣고, 방문 여부를 체크합니다.

i 1 2 3 4 5 6
check[i] true true true False true false

방문 순서 : 1 2

Queue : 2 5 3

Step 3

큐에서 정점 2을 제거하고, 정점 5를 방문합니다. 그리고 인접한 정점 4를 큐에 넣고, 방문 여부를 체크합니다.

i 1 2 3 4 5 6
check[i] true true true true true false

방문 순서 : 1 2 5

Queue : 5 3 4

Step 4

큐에서 정점 5를 제거하고, 정점 3을 방문합니다. 여기서는 인접한 정점들을 모두 방문해서 따로 큐에 추가할 정점은 없습니다.

i 1 2 3 4 5 6
check[i] true true true true true false

방문 순서 : 1 2 5 3

Queue : 3 4

Step 5

정점 3을 제거하고, 정점 4에 방문합니다. 이제 4에 인접한 6을 큐에 추가하고 방문 여부를 체크합니다.

i 1 2 3 4 5 6
check[i] true true true true true true

방문 순서 : 1 2 5 3 4

Queue : 4 6

Step 6

정점 4를 큐에서 제거하고, 마지막 정점 6을 방문합니다. 그리고 큐에 마지막 남은 6이 제거되고 더이상 탐색할 정점이 없으므로 종료합니다.

i 1 2 3 4 5 6
check[i] true true true true true true

방문 순서 : 1 2 5 3 4 6

Queue : 6
