Mybatis LAST_INERT_ID가 1이 나오는 문제

2018-05-21 21:30:00
Issue

iBatis에서는 그냥 기존처럼 값을 받으면, last_insert_id가 반환되었다고 하는데 myBatis로 변경되면서 selectKey에 대해 변경점이 있어서 그렇다고 한다.

위처럼 정의된 상태에서 return되는 Integer 값을 기존 iBatis 방식으로 값을 그냥 받아오면 1만 넘어오는데, 이는 id값이 아니라 insert된 row의 count이다.

리턴되는 값은 파라미터로 넘기는 event에 keyProperty 속성의 값으로 들어가게 된다.

myBatis에서는 insert시 전달받은 parameter에 keyProperty로 정의된 값이나 Object로 자동으로 inject 되게 된다.
